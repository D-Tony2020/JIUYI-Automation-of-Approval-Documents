# 久益供应商承认书自动化 · 技术方案（v1）

> Moore 工业智能 交付物。配套文档：[填写逻辑确认手册](承认书填写逻辑确认手册.md)（15 sheet 逐页已确认）、[spike/](spike/README.md)（抽取流水线已离线验证）。
> 日期：2026-06-21

## 1. 背景与目标
久益是生久的供应商。每当生久下发一份图纸，久益需据此编制一份体量很大的《材料承认书》（生久 Rev3.1 标准，Excel 15 sheet），中间要从图纸抓信息、再到材料库（供应商/我方/三方检验报告、MSDS、UL、图纸等大量非结构化 PDF）找对应材料、提取信息填表。本项目用国产 LLM + 确定性工程，把这一过程自动化成一个 Windows 桌面工具。

**目标**：操作员上传图纸 → 选型确认 → 一键生成可交付的承认书 xlsx（自动部分带来源溯源，人工部分留醒目槽位）。

## 2. 勘测结论摘要（详见手册与记忆）
- 承认书 15 sheet 按填充机理分 5 类：静态模板 / 元数据回填 / 文档装配(OLE) / 图纸驱动+实测(FAI) / **LLM抽取核心(材质成分表)**。
- 做好的承认书内嵌 **44 个可双击打开的 OLE 源文件**（材质表内就嵌 25 份证据）。
- 材质表本质 = **MSDS(成份/CAS/重量%) ⋈ 第三方报告(RoHS十项/编号/日期)** 跨文件 JOIN；报告号/日期可由文件名直取。
- 8 份样本归纳：**BOM 可变、由图纸技术要求驱动**；会出现库外新供应商；材质表是**变行表**；判定栏固定规则；现业务全非车规。
- 五大坑：ND vs 数值、单位/格式换算、命名陷阱、冗余去重、fitz 中文乱码。
- 案例本身存在数据错误（Cu=0.9899 无源、PA66 CAS 取自外部常识）→ **带溯源的自动化可比人工更准**。

## 3. 产品形态：人在环的 5 步工作站
```
①上传生久图纸PDF → 自动解析 BOM+4尺寸+元数据 → 人工复核（源头锚点）
②选型+齐套体检   → 每料类选供应商，要用文件丢进"临时选取池"，系统自动分类初选 → 人工勾选
③抽取+JOIN       → LLM 抽 MSDS ⋈ RoHS → 生成材质表，冲突标红
④装配           → 黄金模板注入 + OLE嵌入(Excel COM)；FAI规格自动，实测/照片=红色待填槽
⑤校验+导出       → 齐套/有效期/未填项检查 → 导出承认书 xlsx（带溯源）
```

## 4. 系统架构
| 层 | 职责 | 选型 |
|---|---|---|
| 输入 | 图纸 PDF（人工上传）、材料临时选取池 | — |
| 解析 | 文本层/表格/栅格化分流 | pdftotext -layout(报告) / pdfplumber(MSDS窄表+CAS修复) / fitz栅格化(扫描件→视觉) |
| 抽取 | 非结构化→结构化(JSON schema 强约束) | **国产 VL 旗舰**：Qwen3-VL + GLM-4.6V 双跑A/B取胜者；模型名配置化；扫描件 VL+专用OCR交叉校验 |
| 主数据 | 去重/别名/有效期/冲突检测 | SQLite；主键=报告号+样品+实验室 |
| 装配 | 黄金模板注入 + OLE嵌入 | openpyxl(数据/变行) + **Excel COM(OLE+EMF预览)** |
| 交付 | 桌面应用 + 溯源面板 + 导出校验 | Windows 桌面（符合久益基线） |

**LLM 只在 4 处出手**：图纸解析、MSDS 抽取、RoHS 抽取、扫描件 OCR。**选型/JOIN/换算/排版/CP-CPK/校验全部确定性代码**。

## 5. 关键技术决策（已定）
- 自动化边界：全本尽可能自动；FAI 实测/实物照片/单重 = 人工槽位（红色待填、未填禁导出）。
- 供应商选型：每次人工勾选 + 齐套体检（库有冗余候选，且会有库外新供应商）。
- LLM：云端国产、**性能优先不看价格**（单份承认书 token 量级几万~十几万，≈¥1）；调用 `trust_env=False` 绕 Clash 代理。
- 输出：黄金模板注入（非 LLM 生成 Excel）；**装配源文件保持 OLE 可打开**（→ 运行机需装 Excel 走 COM）。
- 过期证书：**仅预警不拦截**。
- 自制件字段冲突：标红人工裁决（裁决原则待久益）。
- 真值口径：以源文件为准，案例不作唯一标准答案；每个自动字段带 `来源:文件+页码`。

## 7. 风险与对策
| 风险 | 对策 |
|---|---|
| 国产 LLM 抽取准确率未知 | **spike 锡丝已搭好**，配 key 即可 A/B 出硬数据；再平移到难样本 |
| MSDS 窄表 CAS 截断 | pdfplumber 表格 + CAS 修复（已验证）/ 视觉模型兜底 |
| ND 误填、单位混乱 | 逐项读取 + 确定性换算（已固化于 assemble） |
| OLE 装配依赖 Excel | 运行机装 Excel 走 COM；或降级图片模式（已留选项，老板选 OLE） |
| 自制件字段冲突 | 标红人工裁决；裁决原则待久益 |
| 无车规样本 | 默认非车规；首单车规时补样本细化 |
| 库外新供应商 | "临时选取池"输入方案天然兼容 |

## 8. 路线（里程碑）
1. **M0 spike**（进行中）：锡丝一行打穿 + Qwen/GLM A/B（待 key）。→ 出"国产 LLM 行不行"硬结论。
2. **M1 单料类闭环**：图纸解析→选型→材质表→装配，跑通一个完整品号（建议用 YY60039403 全配对样本）。
3. **M2 全本生成**：15 sheet 全自动 + 人工槽位 + 校验导出。
4. **M3 工程化**：主数据库、桌面外壳、PyInstaller 打包、久益现场试用。

## 9. 待久益确认

3. 首单车规时补车规样本。
4. 运行机是否都装有 Excel（OLE 装配前提）。
