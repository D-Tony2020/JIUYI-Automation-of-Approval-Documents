# 本单输入框架（驱动 UI「本单输入」用户流 + 齐套体检，可复用）

> 回答："本单源文件是否充分填满所有动态单元格？" —— **充分且闭环**。
> 依据：把做好的承认书 YY60039403 实际调用的 **44 份 OLE 源文件**，按"喂给哪块内容"反推归类得到本框架。
> 关键前提：承认书的全部动态内容 = 图纸驱动 + 材质表(MSDS⋈报告) + 装配族(OLE附件)，三者的源文件**就是这 44 份**，故一定闭环。

## 一、输入分两级
- **整品级**（每单固定几份）：图纸 / 包装规范 / 出货报告 + 人工(FAI实测/样品照片)。
- **每料级**（随 BOM，每种材质一套）：MSDS / RoHS / REACH·SVHC / UL / 信赖性·盐雾 / 部件承认书·规格书。

## 五、两个关键设计（影响引擎与 UI）
1. **一份上传一次、系统路由到多处**：同一 MSDS 同时喂"材质表 cell 抽取"和"材质证明书 OLE"；同一报告可同时是"材质表证据 OLE"和"RoHS 数据源"。UI 让用户**按文件上传一次**，引擎按"用途路由"分发到所有需要它的 sheet/cell。
2. **必须传"本单实际文件"，不能套案例材料**：嵌入的是 `磷铜端子MSDS新`/`双鸿油墨` 等本单版本，与散落案例材料文件名/版本不同 → UI 输入流以**用户本单上传**为准；案例材料仅作开发样本。

## 六、UI「本单输入」用户流（5 步，复用本框架）
1. **上传生久图纸** → 解析 BOM/料号/版本/尺寸 → 人工复核。
2. **选型确认**：按 BOM 列出每种材质，人工勾选供应商（库有冗余候选）。
3. **每料上传源文件**：按"第三节矩阵"为每种材质提示需传的文档类型；拖入临时选取池，系统自动分类+初选+人工复核。
4. **上传整品级**：包装规范、出货报告。
5. **人工槽位**：FAI 实测、样品照片。
→ 每步实时**齐套体检**：缺某材质的必需文档类型 → 红色提示"未齐套，不可生成"。

## 七、齐套体检规则（充分性自检）
对每种材质，按第三节矩阵核必需文档类型是否齐；整品级 3 份是否齐；人工槽位是否填。**全齐才放行生成**；缺件红色列出、指明缺哪料的哪类文档。
