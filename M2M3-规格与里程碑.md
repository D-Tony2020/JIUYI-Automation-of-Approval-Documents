# 久益承认书自动化 — M2/M3 规格与里程碑

> 本文是 2026-06-25 grilling 会话的固化产出，定义 **M2（真实输入管线 + GUI）** 与 **M3（产品化交付）**。
> M1（装配引擎）已由 7 案重构验证通过。本规格的每条裁决均有实证支撑，证据见正文。

---

## 0. 边界：M1 给了什么，M2/M3 补什么

**M1 = 装配引擎**：给定结构化输入（BOM / FAI尺寸 / OLE源文件+表归属 / 照片），产出 15 表承认书（段一 openpyxl 填 cell + 段二 COM/WPS 嵌 OLE）。7 案 E2E 6/7 绿（1 案 golden 损坏）。

**但 M1 的输入全是从成品 golden 反抽的** —— 真实新单没有 golden。

**M2/M3 = 真实输入侧**：操作员手上只有〔生久生图纸 + 一堆杂乱供应商材料 PDF + 样品照片〕，要自动识别 / 抽取 / 路由 / 审核，最终装配成承认书。

## 1. 输入管线裁决（A1 / A2 / B / C）

### A1　图纸 → 品号/版本/名称 + FAI 尺寸
- **引擎**：`qwen-vl-max`（阿里云 DashScope 官方端点，红线完全授权；**在线**）
- **品号/版本/名称**：读标题栏（实证：SB/SD/SK 生久内部码 + 版本全读对）
- **FAI**：抽 `标称±公差` callout → **公差归一**（图纸记法 `35₀⁻³` / `98±5` → FAI 三列 中心/USL/LSL）→ 灌**确认环①**人核
- **验证**：7 案多模态盲测，忠实率 **~100%**
  - 030496 扫描/曲线图（regex 零命中）→ 视觉读对 2/2
  - 038296"缺 33.5±1.5" = 等价记法（图纸 `35₀⁻³` = 区间[32,35] = golden 重心化的 33.5±1.5）
  - 039397"缺" = **golden 损坏**（嵌了 402 的图），非模型错
- **红线**：仅官方第一方端点、不进训练、不经第三方中转

### A2　BOM 骨架（零件 → 材质 → 材质类别 → 供应商）
- **引擎**：**GUI 人工填**（供应商=采购事实、材质类别=装配事实，图纸上物理不存在，多模态也救不回）
- **交互**：**脊柱优先 + 边键边联想匹配**（操作员键料名，系统即从 PDF 堆高亮匹配文件供一点确认）
- 是整张承认书的**脊柱**，B/C 都挂其上

### B　材质成分（成份 / CAS / 重量% / RoHS）
- **引擎**：材料 PDF → `qwen`（官方）自动抽取
- **验证**：盲测 **97% CAS 召回**（源完整时；瓶颈在源完整性非抽取）
- 抽出后挂到 A2 脊柱对应料行
- **⭐片内定位 = 内容驱动（M2.1 实证修正）**：B 开 PDF 读内容即知"这份是哪个料"，**"文件→料行"链路从内容白来、可靠**；让 B 那趟 LLM 顺带返回料identity → 直接驱动 材质表片内落位（料行×K/L/Y）。
  - **文件名模糊匹配不可靠**（实证 [`hitl/file_match.py`](hitl/file_match.py) 锚点403）：别名泄漏（"锡"⊂"镀锡铜"→07CU无铅锡线误判镀锡铜）、颜色加成压过料名（热缩管黑色→误判黑色油墨）、纯色号命名（红-CANEC…）无 token。**仅作 fallback/预筛**，最终由确认环②人工确认。

### C　文件 → OLE 槽路由
- **引擎**：文件名关键词 + 扇出规则**自动分类（~99%）** + 树形图人工兜底
- **验证**：7 案 221 文件，naive 规则 92% → 补 trivial 规则 ~99%，硬残 ~1%（供应商料号乱名如 `EN-TU5601-401D003`）
- **扇出**：1 文件 → N 槽（同一份 MSDS 同时进 材质表 + 材质证明书，重叠 50–100%）→ 树形图须支持一文件挂多节点
- **二级定位**：材质表内还要按料行落到 K/L/Y 列（MSDS/REACH/RoHS）

## 2. 操作流（5 步 + 两确认环）

| 步 | 动作 | 引擎 / 门禁 |
|---|---|---|
| 0 | 顶栏配置：操作员 / 通用模板 / 输出目录 | 复用 settings.json |
| 1 | 新建工单 **+ 上传图纸** → 抽 品号/版本/FAI | qwen-vl（在线）→ **确认环①「图纸识别」** |
| 2 | **搭 BOM 骨架**：键 零件→材质→材质类别→供应商 | A2 人工（脊柱）+ 边键边联想匹配 |
| 3 | **拖入材料 PDF 堆** → 路由 OLE 槽 + 抽材质成分 | C 自动~99% + B 自动97% → **确认环②「文件树+材质」** |
| 4 | 贴样品照片（2–4 张剪贴板） | 保宽高比布局 |
| 5 | **总装 + 导出**：段一 cells + 段二 OLE | COM/**WPS** → 溯源面板 + 有效期预警 + **未填禁导出** → 承认书.xlsx + 打印 |

### 确认环（4 决策已定）
**分步两环 / 示意空槽图 / 可改实时重算 / 允许人工标豁免**
- **环①图纸识别**：FAI/品号 可视化 + 改 + 豁免
- **环②文件树+材质**：OLE 槽位树形图（示意空槽）、自动路由可核对 + 拖拽纠错、材质成分核对、豁免标记

## 4. 验证策略：破循环（伪真单）

现有 `run_e2e` 是**循环论证**：从 golden 反抽输入 → 拼回 → 比 golden（只验装配引擎，不验真实输入侧）。

**M2 改用伪真单（pseudo order）**：
- 从案例拆出**纯原始输入**：raw 图纸 PDF / raw 材料 PDF 堆（原始名、打散、**剥掉表归属**）/ 照片
- **藏掉 golden 的表归属与 FAI**
- **C 分类器 + FAI 多模态真正接入**，不许碰 golden 拐杖
- golden 仅作最终对标
- 这是 M2 所有"无拐杖"测试的基准
