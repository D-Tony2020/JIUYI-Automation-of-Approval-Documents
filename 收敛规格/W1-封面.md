# W1·封面 — 业务收敛规格

> Phase A 产出物。状态：⏳待老板签认（签认前不写填充码）。
> 锚点案例：YY60039403 承认书「封面」sheet。空白模板：`模板\承认书空白模板.xlsx`。

## 实勘结论（先把最重要的事实摆出来）

空白模板「封面」里，**只有 3 个格是空的、需要工具填**：`D12 / D14 / D16`。
其余全部是**模板静态内容**（已预填，工具不碰）：签字人 C21/D21/E21=Leven/王立均/张启宇、供应商地址 C27、电话 C29、邮箱 C33、Logo、以及所有标签 B4/B10/B12/B14/B16/B19/B20…。

---

## 1. 填哪些格（仅 3 格）

| 格 | 合并区 | 字段 | 案例值 |
|---|---|---|---|
| D12 | D12:E13 | 材料名称（品类词） | 导线 |
| D14 | D14:E15 | 生久料号 | YY60039403 |
| D16 | D16:E17 | 图纸版本 | A01 |

> 合并单元格 → 值写左上格（D12/D14/D16）。

## 2. 空白模板长啥样（初始结构）

- **已静态预填**：B4 宁波生久科技有限公司 / B10 材料承认书 Rev3.1版本 / 标签 B12·B14·B16·B19·F19·B20·C20-E20·B26·B28·B30·B32 / **签字人 C21·D21·E21** / **供应商档案 C27·C29·C33** / Logo@B1。
- **空待填**：仅 D12·D14·D16。
- 无人工实测红槽。

## 3. 数据从哪来

3 格全部来自**本单生久图纸的元数据**（HITL 阶段由人录入/复核的图纸标题栏字段，自动图纸解析延后到 W3/M2）：
- D12 ← 图纸「名称 TITLE」里的**品类词**（如 SB120420BLCNR009**导线** → 导线）
- D14 ← 图纸「品号 MATERIAL NO.」（直取）
- D16 ← 图纸「版本号 REV」（直取）

## 4. 数据怎么处理

- D14、D16：**直取**，零处理。
- D12：从图纸名称**切出品类词**。名称形如「<图号><品类词>」，取尾部品类词。MVP 先支持「导线」，品类词典做成可扩展配置（导线/扇框/扇叶/漆包线/线架…）。

## 5. 什么样是对的（验收=自检依据）

- D12=="导线"、D14=="YY60039403"、D16=="A01"（对标 YY60039403）。
- 3 格写入后非空；写入的是合并区左上格。
- 其余静态格**值不被改动**；字体/对齐/合并区不变。

## 6. 怎么自检（脚本断言）

- openpyxl 读回 `D12/D14/D16` == 期望值。
- 读回 `C21/D21/E21/C27/C29/C33/B4/B10` 等静态格 == 模板原值（未被误改）。
- zip 级：无新增 `xl/externalLinks`；media 不膨胀。
- 溯源：3 格各带 `来源:图纸元数据.<字段>`。

## 7. 可能哪些坑

- 品类词提取：非「导线」产品（扇框/漆包线）时切词可能错 → 词典外要拦。
- 合并单元格：必须写左上格，写到 E12/E13 之类被合并的格会丢。
- 锚点错则全表错：D14 料号 / D16 版本来自图纸元数据，若上游录错，封面+FAI+材质表全错（故依赖 §10 的图纸复核门禁）。
- 图纸名称含图号前缀，切品类词不能误把图号带进去。

## 8. 怎么提醒用户

- D12 品类词不在词典 → 弹「未识别品类词，请确认」让人工填/选。
- D14/D16 与 HITL 录入的图纸元数据不一致 → 提示核对（理论上同源，做一致性自检）。

## 9. 哪些手动填 👤

- 无。封面无人工实测槽位。

## 10. 哪些人工介入

- **上游**：图纸元数据复核（人录入/确认 品号·版本·名称），封面消费其结果。这是 W3 的门禁，封面 §3 的三个源值由它产生。
- D12 品类词词典外时人工确认。

---

## 最小执行线（按"数据源/规则"分组）

**唯一一条线** = 「图纸元数据 → D12/D14/D16」。一个数据源（图纸元数据）经其规则（品类词提取 + 直取）落到 3 个可独立验证的格 = 封面整表一条线。
