# 图纸识别确认环 — 业务收敛规格（用户流第①环）

> Phase A 产出。状态：✅ 四项关键设计已老板签认（2026-06-25）。这是"上传图纸→人工确认识别结果"的 HITL 门禁，与"供应商选择+上传"同级、在其前。

## 1. 流位（已定：分步两环）
```
上传审图图纸 → 【① 图纸识别确认环】 → 【② 供应商选择 + 源文件上传】 → 生成
```
- ① 与 ② 是**两个相邻同级环**；① 是 ② 的**前置门禁**：先把图纸推出来的结构摆给人确认、锁定，再进选型/上传。纠错最早、流程清晰。

## 2. 呈现内容（一屏四块）
1. **整品识别卡**：品号 / 品名 / 版本 / 品类词(D12) / 图纸尺寸(n±t 列表→FAI 规格)。
2. **BOM 识别树**：零件 → 材质(类别)；标注每个零件是"采购件"还是"工艺料"（锡=工艺料，不出部件承认书/UL）。
3. **各页 OLE 槽位概览**：每页应有槽位数 + 驱动量（见 §3）。
4. **各页 OLE 空槽可视化**（已定：**示意空槽图**）：每页 N 个空槽按其网格位摆开，每槽标"待上传：<材质/部件> 的 <MSDS/UL/证明>"。直观看到该传什么、几个、怎么排。轻、快。

## 3. 槽位计数规则（来自 7 案实证，见〈嵌入组结构搭建〉〈材质表结构搭建〉）
| 页 | 槽位数 |
|---|---|
| 材质成分展开表 | = 材质数（块/色更细分） |
| 材质证明书 | = 材质数（一材一证） |
| 供应商部件承认书 | = 采购部件数（零件去工艺料） |
| UL 证明 | = 部件承认书数（每采购件一张） |
| 信赖性 | ≈ 部件数（松，参考） |
| 图纸 / 包装 / 出货 | 恒 = 1 |

## 4. 交互与门禁（全部已定）
- **可编辑（已定：可改·实时重算）**：本环可**增删材质/部件、改品号/尺寸**；计数与各页槽位**实时跟着重算**。识别错就地改，是 HITL 纠错核心。
- **标注纪律**：🟡黄=自动识别·可改；🔴红=需人工裁决（如"锡是工艺料不出部件承认书""某尺寸单边公差"）；虚线框=待人工上传的空槽。
- **签认门禁**：未点"确认识别结果"**不可进 ②**；确认后**锁定结构**（BOM/计数），再进选型/上传。
- **衔接 ②**：确认后每材质一行选供应商；**每个 OLE 空槽 = 一个上传位**，"一份上传系统路由到多处"（沿用〈本单输入框架〉）。
- **样品照片上传（② 内，已定）**：用户从微信收图 → **剪贴板粘贴**进 UI（保留该习惯），传 **2-4 张**；**不在 Excel 里标红**。Excel 布局**保留原图宽高比、只缩放不变形**（实证 7 案：小照=竖图 0.6-0.8 细节、长照=横图 1.1-4.0 整体）：**竖照同高侧排在上排、横照满宽限框堆叠在下方**，由 `hitl/sample_photo.fill_sample_photo` 按每张原图比例动态算尺寸。LOGO 保留、不重叠、绝不压扁。
- **齐套校验**：用 §3 计数当基线，实时"已传 X / 应有 N"；**实测 < 槽位数 → 标'漏嵌'**；未齐套禁导出。
- **源缺豁免（已定：允许）**：某料确无成分 MSDS（如案例 YY60039397 只嵌 RoHS 报告），人工勾"本件无此源"→ 齐套校验对该槽**放行**、不卡导出；豁免留痕。

## 5. 什么样是对的（验收）
- 上传图纸后，整品/BOM/各页槽位数**自动摆出**且与图纸一致；黄/红/虚线标注分明。
- 增删一个材质 → 材质证明书槽位 +1/−1、材质表行数随之变（实时）。
- 锡被识别为工艺料、不进部件承认书/UL（红标待裁，可人工改判）。
- 未确认时"下一步/上传"锁死；确认后解锁 ②。
- 齐套面板：漏嵌红标、可豁免、未齐套禁导出。

## 6. 边界 / 归属
- 本环是 **M2（自动图纸解析）的人审界面**：M1 期人工在此**录入**结构，M2 期自动解析**预填**、人工**复核**——同一界面、前向兼容。
- 槽位计数/布局的算法已固化（`study/embed_structure.py`、`hitl/material_table.compute_layout`）；本环是其**可视化 + 人审**外壳。
- UI 实现属框架层，待 W8 总装后或 M2 接入时建；本规格为其蓝本。
