# W3·图纸（2.图纸 生久&供应商） — 业务收敛规格

> Phase A 产出物。状态：⏳待老板签认。**第一张装配族（OLE）表**，性质与 W1/W2（填格）根本不同。

## 实勘结论
- 交集工具（7 样本）：图纸表只有 **3 个静态格**（Logo/标题/页脚网址），**零动态单元格**。
- 动态内容 = **嵌入的 1 个 OLE = 本单生久图纸 PDF**（不是格值，交集工具照不到）。

## 1. 填什么（无格可填，嵌 1 个 OLE）
- 在图纸表嵌入 **1 个 OLE 对象** = 本单生久图纸 PDF（整份，可双击打开）+ **PNG 预览缩略图**（首页渲染，Excel 自动转 EMF）。
- "& 供应商"位留空（久益不出转化图，W1 已确认）。

## 2. 空白模板长啥样
- 图纸表初始：Logo + 标题"图纸（生久&供应商）" + 页脚网址，**无 OLE**。OLE 锚点位置对齐 golden（精确 Left/Top 建 W3 时从做好的承认书取）。

## 3. 数据从哪来
- 本单**生久图纸 PDF**：HITL 每单人工上传（W1 已确认图纸入口）。
- 本单（YY60039403）demo：从做好的承认书内嵌 OLE 抽取（= `D:\tmp\sheet2_drawing.pdf`，图号 SB120420BLCNR009导线），落进项目 `本单输入/` 作输入。

## 4. 数据怎么处理（复用 spike/ole_embed.py 轮子）
- `make_icon(pdf)` → fitz 渲染首页为 PNG 预览。
- `com_session()` → DispatchEx Excel，四必设（Visible/DisplayAlerts/AskToUpdateLinks/Interactive=False）+ 子进程 + 退出强杀。
- `embed(ws, src=图纸PDF, png=预览, Left, Top)` → `OLEObjects().Add(ClassType="Package", Filename, IconFileName, DisplayAsIcon)`。

## 🔑 架构关键（本表确立、影响全局）
**两段式，不可破**：
1. **段一 openpyxl** 填完所有"格"（W1/W2/W4/W5…的 cell 填充，build_upto）。
2. **段二 COM** 嵌入所有 OLE（W3 图纸 + W6 装配族 + W8）。**OLE 嵌入是终态**——一旦嵌了 OLE，**绝不能再用 openpyxl 保存**（会丢掉全部 OLE）。
- 故 OLE 类表的"填充"不进 openpyxl 的 build_upto 流水线；它们是流水线之后的 COM 段。
- 运行机**必须装电子表格 COM 引擎**——**客户机=WPS（已实证兼容）**：WPS 提供 Excel 兼容 COM `KET.Application`，两段式代码一字不改跑通；`com_session` 优先绑 WPS、回退 Excel。

## 5. 什么样是对的
- 图纸表含 **1 个 OLEObject**；双击能打开本单图纸 PDF；预览显示图纸首页缩略图；位置对齐 golden；3 静态格不动。
- Excel 复开**无修复弹窗、无外链提示**。

## 6. 怎么自检
- zip 级：`xl/embeddings/oleObject*` 计数 = 段一基数 + 1。
- COM 复开：`OLEObjects().Count == 1`，打开无异常（spike 的 verify 模式）。
- 静态 3 格值不变；无外链。
- ⚠️ openpyxl 的 cell-diff 不适用（OLE 非格值）；改用 **OLE 计数 diff（0→1）**。

## 7. 可能哪些坑
- openpyxl 再存丢 OLE（架构已规避：COM 段终态）。
- COM 无头挂死/僵尸 EXCEL → 子进程硬超时 + PID 强杀（spike 已含）。
- 预览 PNG 清晰度（dpi）/锚点压字 → 调 dpi 与 Left/Top。
- 图纸 PDF 路径含中文空格 → COM Filename 需绝对路径。

## 8. 怎么提醒用户 / 9. 人工填 / 10. 人工介入
- 人工介入：**上传本单生久图纸 PDF**（W3 入口）；图纸元数据复核（W1 已用）。
- 无红槽人工填。
- 提醒：未上传图纸 PDF → 硬拦截不嵌。
